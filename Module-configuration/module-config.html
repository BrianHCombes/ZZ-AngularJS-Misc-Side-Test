• The original reason this turned into a side test is I had a terrible time getting the module to work when configured as a separate file named module-config-service.js.

• Only to find out it was to make sure the module-config-controller.js file is listed before the 'module-config-service.js' file in the default index.html file (called module-config.html in this case) because the 'module-config-controller.js' is where the module is defined per the '[]' brackets. Thus, the 'module-config-controller.js' file has to be loaded first.
